## Task A: Ingestion Handler

**Source:** Original Task 8 from v1 plan (removed during v1 pre-implementation review).

**Files:**
- Create: `mcp/src/ingest.ts`
- Create: `mcp/src/ingest.test.ts`
- Modify: `mcp/src/server.ts` â€” start Unix socket listener on `~/.memtree/<hash>/ingest.sock` after DB open; pipe incoming lines to `processIngest`; close socket on shutdown

The ingestion handler listens on a Unix socket (`~/.memtree/<hash>/ingest.sock`), accepts JSON payloads from `capture.sh`, normalizes them, applies redaction, and writes qualifying content to the store as `file_chunk`, `grep_result`, or `bash_output` nodes.

Payload schema:

```typescript
export interface IngestPayload {
  tool: 'Read' | 'Grep' | 'Bash';
  input: Record<string, unknown>;
  response: Record<string, unknown>;
  session_id: string | null;
  cwd: string;
  ts: number;
}
```

Key invariants:
- `shouldDropPath` rejects Read payloads matching denylist
- `shouldDropBashCommand` + `redactBashOutput` applied to Bash payloads
- **Per-project capture opt-out:** honor `config.capture.read`, `config.capture.grep`, `config.capture.bash` boolean flags â€” if a flag is `false`, drop the payload before any processing
- **Cheap-filter fast path:** if content length â‰Ą `config.capture.filterMinSize` AND `content_hash` is not found in the last-60s dedup window â†’ insert node directly as `status='live'`; only payloads needing expensive future checks (embedding similarity dedupe) go through `status='pending'`. This is the common path â€” most nodes should land as `live` immediately.
- **Minimum content size gate:** configurable, default 50 bytes (`config.capture.filterMinSize`) â€” drop payloads below this
- **Dedup window:** hash(`tool` + `JSON.stringify(input)`) â€” skip if same hash seen in last 60 s
- **Bounded ingestion queue:** the socket ingestion loop feeds a fixed-size in-memory ring buffer (default 1,000 entries) consumed by a prepared-statement `INSERT` loop. If the buffer fills (server pinned), excess writes are **dropped with a `process.stderr.write` warning** â€” never block or back-pressure the hook. The hook is fire-and-forget by design.
- **`metadata.gitignored` flag:** for Read/Grep payloads, check whether the file path is gitignored (run `git check-ignore -q <path>` from `cwd`; cache results for 30 s). If gitignored, set `metadata.gitignored=true` on the node row. Gitignored â‰  secret, but useful signal for the filter walker. Do not drop gitignored paths (path denylist is the drop gate, not gitignore).
- **Filesystem hardening:** `~/.memtree/<hash>/` directory created at mode `0700`; `ingest.sock` file at mode `0600`. Applied in `server.ts` when the socket is bound.

## Task B: Capture Hook Script

**Source:** Original Task 15 from v1 plan (removed during v1 pre-implementation review).

**Files:**
- Create: `mcp/hooks/capture.sh`

Shell script invoked by the PostToolUse hook. Reads `$CLAUDE_TOOL_NAME`, `$CLAUDE_TOOL_INPUT`, `$CLAUDE_TOOL_RESPONSE` from env, constructs JSON payload via `jq`, and sends to the Unix socket via `socat`.

Hard deps added in v1.1: `jq â‰Ą1.6`, `socat â‰Ą1.7`

**Project-hash lookup uses `projects.tsv`** (tab-separated `cwd\thash`) to avoid spawning a Node process per hook call. The TSV is written by `registerProject()` at server startup.

**Walk-up algorithm:** the hook cannot assume `$MEMTREE_CWD` matches a `projects.tsv` entry exactly â€” the file that triggered the hook may be in a subdirectory. Walk up the directory tree from `$MEMTREE_CWD` toward `/`, checking each candidate path against the first column of `projects.tsv`, until a match is found or `/` is reached with no match (in which case exit 0 silently).

**`session_id`:** Claude Code exposes `$CLAUDE_SESSION_ID` in the PostToolUse hook environment. Pass it through. If the env var is absent (e.g. older Claude Code versions), fall back to `null`.

```bash
#!/usr/bin/env bash
set -euo pipefail

# TODO(v1.2): replace this shell pipeline with a compiled hook binary

# Walk up from $MEMTREE_CWD to find nearest registered project root
HASH=""
DIR="${MEMTREE_CWD:-$PWD}"
PROJECTS_TSV="${HOME}/.memtree/projects.tsv"
while [[ "$DIR" != "/" ]]; do
  HASH=$(awk -F'\t' -v cwd="$DIR" '$1==cwd{print $2;exit}' \
    "$PROJECTS_TSV" 2>/dev/null || true)
  [[ -n "$HASH" ]] && break
  DIR="$(dirname "$DIR")"
done

[[ -z "$HASH" ]] && exit 0

SOCKET="${HOME}/.memtree/${HASH}/ingest.sock"
[[ ! -S "$SOCKET" ]] && exit 0

SESSION_ID="${CLAUDE_SESSION_ID:-null}"
# Wrap session_id as a JSON string unless it's already null
if [[ "$SESSION_ID" != "null" ]]; then
  SESSION_ID="\"${SESSION_ID}\""
fi

jq -cn \
  --arg tool  "$CLAUDE_TOOL_NAME" \
  --arg cwd   "$MEMTREE_CWD" \
  --argjson session_id "$SESSION_ID" \
  --argjson input    "${CLAUDE_TOOL_INPUT:-null}" \
  --argjson response "${CLAUDE_TOOL_RESPONSE:-null}" \
  '{tool:$tool,input:$input,response:$response,session_id:$session_id,cwd:$cwd,ts:(now*1000|floor)}' \
| socat - "UNIX-CONNECT:${SOCKET}" &>/dev/null || true
```

## Task K: `memtree.bash` Tool

**Files:**

- Create: `mcp/src/tools/bash.ts`
- Create: `mcp/src/tools/bash.test.ts`
- Modify: `mcp/src/server.ts` â€” register `memtree_bash` tool handler
- Modify: `mcp/src/config.ts` â€” add `trustedExecution?: boolean` to `MemtreeConfig` (default `false`)

Executes a shell command by routing through Claude Code's native `Bash` tool via `mcp-exec`, inheriting its permission prompts and sandbox surface. Applies command denylist + output redaction before storing the result.

**Default mode is passive-only** (store-and-recall, no execution): `memtree.bash` without `trustedExecution: true` throws a clear error. Set `trustedExecution: true` in global or project config to enable in-process invocation via `mcp-exec`.

```typescript
import { mcpExec } from '@anthropic/mcp-exec';
import { shouldDropBashCommand, redactBashOutput } from '../redaction';
import { insertNode } from '../store/nodes';

export interface BashParams {
  command: string;
  budget_tokens?: number;
}

export interface BashResult {
  nodeId: string;
  stdout: string;
  stderr: string;
  exit_code: number;
  truncated: boolean;
}

export async function memtreeBash(
  db: Database.Database,
  config: MemtreeConfig,
  params: BashParams
): Promise<BashResult> {
  const { command, budget_tokens = 2000 } = params;

  if (shouldDropBashCommand(command)) {
    throw new Error(`Command rejected by denylist: ${command}`);
  }

  if (!config.trustedExecution) {
    throw new McpError(ErrorCode.InvalidParams,
      'memtree.bash requires trustedExecution: true in config. ' +
      'Set this only in trusted local environments.');
  }

  // Route through mcp-exec â†’ Claude Code's native Bash (sandbox + permissions)
  const raw = await mcpExec('Bash', { command });

  const redactedStdout = redactBashOutput(raw.stdout ?? '');
  const redactedStderr = redactBashOutput(raw.stderr ?? '');
  const content = redactedStdout || redactedStderr;
  const originalBytes = Buffer.byteLength(content, 'utf8');
  const truncated = originalBytes > config.capture.maxBytes;
  const stored = truncated ? content.slice(0, config.capture.maxBytes) : content;

  const nodeId = ulid();
  if (stored.length >= config.capture.filterMinSize) {
    insertNode(db, nodeId, {
      parent_id: null,
      kind: 'tool_output',
      source_uri: `tool:Bash#${createHash('sha256').update(command).digest('hex').slice(0, 8)}`,
      content: stored,
      content_hash: createHash('sha256').update(stored).digest('hex'),
      status: 'live',
      mtime: 0,
      truncated: truncated ? 1 : 0,
      original_bytes: originalBytes,
      metadata: JSON.stringify({ tool: 'Bash', command, exit_code: raw.exit_code }),
    });
  }

  const lines = applyBudget(stored, budget_tokens);
  return { nodeId, stdout: lines, stderr: redactedStderr, exit_code: raw.exit_code ?? 0, truncated };
}
```

Tests:

- Command denylist rejects `env`, `printenv`
- Output redaction removes AWS keys / GitHub tokens before storage
- DB row is written for a normal command (trustedExecution=true)
- `trustedExecution=false` (default) throws the documented error
- `mcp-exec` errors propagate (don't swallow)

```typescript
export function loadProviders(config: MemtreeConfig): {
  embedding: EmbeddingProvider | null;
  summarizer: SummarizerProvider | null;
} {
  const embedding = config.embeddingModel
    ? config.embeddingModel.startsWith('openai/')
      ? new OpenAIEmbeddingProvider(config.embeddingModel.slice(7))
      : new OllamaEmbeddingProvider(config.embeddingModel)
    : null;

  const summarizer = config.summarizerModel
    ? config.summarizerModel.startsWith('claude-')
      ? new AnthropicSummarizerProvider(config.summarizerModel)
      : config.summarizerModel.startsWith('openai/')
        ? new OpenAISummarizerProvider(config.summarizerModel.slice(7))
        : new OllamaSummarizerProvider(config.summarizerModel)
    : null;

  return { embedding, summarizer };
}
```

### Provider outlines

**`OpenAIEmbeddingProvider`** â€” `openai` SDK `embeddings.create`; `text-embedding-3-small` default. Requires `OPENAI_API_KEY`.

### Indexing strategy

SQLite's generated columns allow indexing into JSON without a full scan:

```sql
ALTER TABLE nodes ADD COLUMN meta_tool TEXT
  GENERATED ALWAYS AS (json_extract(metadata, '$.tool')) STORED;
ALTER TABLE nodes ADD COLUMN meta_session_id TEXT
  GENERATED ALWAYS AS (json_extract(metadata, '$.session_id')) STORED;
ALTER TABLE nodes ADD COLUMN meta_gitignored INTEGER
  GENERATED ALWAYS AS (json_extract(metadata, '$.gitignored')) STORED;

CREATE INDEX idx_nodes_meta_tool ON nodes(meta_tool) WHERE meta_tool IS NOT NULL;
CREATE INDEX idx_nodes_meta_session_id ON nodes(meta_session_id) WHERE meta_session_id IS NOT NULL;
CREATE INDEX idx_nodes_meta_gitignored ON nodes(meta_gitignored) WHERE meta_gitignored = 1;
```
